The core idea: the generator LLM is never called until the retrieved context passes an assessment gate. A model that isn't invoked on bad context cannot fabricate from it. Everything else serves that gate.

Inside the gate

This is the self-correction. Four checks run cheapest-first; the generator only fires on a clean pass.

1 · Sufficiency
Max retrieval similarity vs threshold. Too low → INSUFFICIENT before any LLM call. (0 tokens)
2 · Triple extraction
Facts as (subject, relation, object) with qualifiers + bi-temporal validity windows.
3 · Clash detection
Same subject+relation, overlapping validity, different value → contradiction. Evidence-voted, OCR-aware.
4 · Answerability
SUFFICIENT / PARTIAL / INSUFFICIENT / AMBIGUOUS — and proposes clarify options.
✓ Answer
Context clean → generate with a confidence band and chunk-level citations.
⇄ Dual-answer
Conflicting facts → present both with sources. Never silently picks one.
? Clarify
Ambiguous question → multiple-choice, then re-run with the answer.
⊘ Refuse
Not in corpus → decline with gap analysis. No fabrication.

Every conversation, document, and codebase becomes one global graph. Recall is O(neighborhood), not O(history) — and it persists across chats and models via MCP.
