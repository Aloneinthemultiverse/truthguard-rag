The problem

Four ways RAG lies to you

Retrieval-augmented generation fails quietly. The answer looks confident and cites a source — but the source was wrong, missing, contradicted, or misread.

P1
Blindness
Scanned pages have no text layer. The retriever never sees them — so the fact simply doesn't exist.
P2
Omission hallucination
The answer isn't in the corpus at all, so the model invents one from partial context.
P3
Silent arbitration
Two documents disagree. The model silently picks one and never tells you there was a conflict.
P4
Ambiguity collapse
The question had two meanings. The model answers one and pretends the other never existed.

How it works

The generator never fires on bad context

A model that isn't invoked on bad context cannot fabricate from it. Everything in the pipeline serves that one gate.

01
Ingest
Mixed PDFs, scans, code-in-PDF → Markdown. No text layer → OCR ladder with per-word confidence.
02
Retrieve
Three signals scored in parallel — vectors, keywords, entity matches — fused and reranked.
03
Assess gate
Before any answer: sufficient? contradictory? ambiguous? Cheapest checks run first.
04
Respond
Only now does the generator fire — or the system refuses, clarifies, or answers both ways.
Answer
Clean context → generate with a confidence band and chunk-level citations.
Dual-answer
Conflicting facts → show both with their sources. Never silently picks.
Clarify
Ambiguous question → ask which one you meant, then re-run.
Refuse
Not in the corpus → decline with gap analysis. No fabrication.

Context memory

One graph for every conversation, document, and repo

Memory isn't a chat log. Conversations, documents, and codebases become one global graph — so recall costs O(neighborhood), not O(history), and it persists across chats, sessions, and models via MCP.
